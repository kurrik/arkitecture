A simple pipeline

Three top-level nodes laid out left to right, joined by arrows. With no anchors given, each arrow auto-routes to the nearest facing edges.

pipeline.ark
webapp {
  label: "Web App"
}
api {
  label: "API"
}
database {
  label: "Database"
}

webapp --> api
api --> database

Nesting & direction

Nodes nest to any depth, and a separate @layout block arranges each parent's children with a direction. Here a vertical Platform stacks a gateway above a horizontal row of services.

nesting.ark
platform {
  label: "Platform"

  gateway {
    label: "API Gateway"
  }

  services {
    label: "Services"

    users   { label: "Users" }
    orders  { label: "Orders" }
    billing { label: "Billing" }
  }
}

@layout {
  platform          { direction: vertical }
  platform.services { direction: horizontal }
}

Bounded contexts with anchors

A Domain-Driven Design sketch: two bounded contexts, a borderless box: none node that groups Cart and Checkout for layout only, and an arrow routed through named anchors and a nested node path.

contexts.ark
ordering {
  label: "Ordering Context"

  row {
    cart     { label: "Cart" }
    checkout { label: "Checkout" }
  }

  orders {
    label: "Orders"
    anchors: [db]
  }
}

inventory {
  label: "Inventory Context"
  anchors: [api]
}

@layout {
  ordering     { direction: vertical }
  ordering.row { box: none; margin: 0; direction: horizontal }

  ordering.orders { anchor db: [0.5, 1.0] }
  inventory       { anchor api: [0.0, 0.5] }
}

ordering.orders#db --> inventory#api
